# テスト/監視（Observability）を整備する

## 背景
品質と運用性を担保するため、自動テストと監視の仕組みが必要。

## スコープ
- コマンドの単体/結合テスト（SQLiteメモリ、モック、ドライラン）
- ロギングの標準化（コンテキスト: job, date, path, size, duration）
- エラー通知の網羅（失敗メール、ログレベル）

## 非スコープ
- 外部監視サービス連携

## 受け入れ基準
- `composer test` で新規テストが安定して通る
- 失敗系テストで通知がトリガーされる

- [ ] ログフォーマットをドキュメント化
- [ ] CIへの組み込み（任意）
